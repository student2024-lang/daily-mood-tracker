# daily-mood-tracker
# Daily Mood Tracker: Вимоги до продукту

## 1. Короткий опис продукту
Daily Mood Tracker — це простий інструмент для щоденного відстеження настрою користувачів.  
Проблема: люди не фіксують свій емоційний стан, забувають про зміни настрою та не розуміють, що впливає на їх самопочуття.  
Для кого: кінцевий користувач — будь-яка людина, що хоче слідкувати за своїм настроєм.  
Користь: дозволяє швидко записувати свій настрій, переглядати історію та помічати закономірності.

---

## 2. Стейкхолдери
1. Кінцевий користувач — отримує цінність від фіксації настрою і можливості бачити історію.  
2. Автор продукту — відповідає за розробку та підтримку MVP.  
3. Викладач / інструктор — оцінює правильність виконання практичної.  
4. GitHub — платформа для зберігання і демонстрації результатів.

---

## 3. Функціональні вимоги (FR)

| ID | Опис |
|----|------|
| FR-01 | Система повинна дозволяти користувачу вибрати свій настрій за день. |
| FR-02 | Система повинна зберігати записи про настрій. |
| FR-03 | Система повинна відображати список всіх попередніх записів. |
| FR-04 | Система повинна дозволяти редагувати або видаляти запис. |
| FR-05 | Система повинна повідомляти користувача про успішне збереження запису. |
| FR-06 | Система повинна мати можливість сортувати записи за датою. |

---

## 4. Нефункціональні вимоги (NFR)

| ID | Опис | Перевірка |
|----|------|-----------|
| NFR-01 | Час відповіді на збереження запису ≤ 2 с | Метрика: тест швидкості |
| NFR-02 | Дані зберігаються коректно та не губляться | Тест: створити, редагувати, видалити |
| NFR-03 | Інтерфейс повинен бути зрозумілим для новачка | Тест: користувач проходить базовий сценарій без інструкцій |
| NFR-04 | Система повинна працювати на смартфоні та ПК | Перевірка: відкриття на різних пристроях |
| NFR-05 | Система повинна захищати дані користувача | Перевірка: доступ сторонніх осіб відсутній |
| NFR-06 | Записи зберігаються мінімум 30 днів | Метрика: перевірка бази даних |
